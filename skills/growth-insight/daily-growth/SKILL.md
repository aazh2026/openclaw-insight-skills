# 每日精进与认知突破

## 核心理念

**每日精进**：基于当日对话、阅读、思考，提炼关键学习点和 actionable insights。

**认知突破**：识别并记录思维模式的升级、认知边界的扩展、心智模型的更新。

## 工作流程

### 1. 收集当日输入

读取以下来源获取当日学习素材：
- 当日 memory 文件 (`memory/YYYY-MM-DD.md`)
- 当日对话历史
- 用户主动分享的学习内容

### 2. 提炼每日精进

从当日输入中提取：

**知识获取**
- 新学到的重要概念/理论
- 有价值的数据/事实
- 行业动态和趋势

**思维升级**
- 对某个问题的新视角
- 原有认知的修正/补充
- 跨学科连接的建立

**实践启发**
- 可立即应用的方法/技巧
- 需要尝试的新做法
- 待验证的假设

### 3. 识别认知突破

判断标准（满足任一即算突破）：
- 🧠 **心智模型更新**：用新模型替代旧模型
- 🔗 **跨域连接**：建立不同领域间的新联系
- ❓ **问题重构**：对问题的定义方式发生改变
- 🎯 **优先级重构**：价值判断标准发生变化
- 🚫 **去伪存真**：识破并摒弃错误认知

### 4. 输出格式

```markdown
# 📚 每日精进 - YYYY年MM月DD日

## 今日核心收获

### 1. [主题1]
**来源**：[对话/阅读/思考]
**要点**：[一句话概括]
**深度**：[详细阐述]
**行动**：[具体可执行的行动]

### 2. [主题2]
...

## 🧠 认知突破

| 领域 | 旧认知 | 新认知 | 触发点 |
|------|--------|--------|--------|
| [领域] | [旧观点] | [新观点] | [什么触发了改变] |

## 🔗 跨域连接

- [概念A] ↔ [概念B]：[连接描述]

## 📋 待办行动

- [ ] [行动1]
- [ ] [行动2]

## 💡 明日思考

[留给明天的思考题或探索方向]
```

## Agent 协作模式

当多个 Agent 同时被召唤时，各自负责：

| Agent | 负责领域 | 输出重点 |
|-------|----------|----------|
| Warren | 投资/商业思维 | 投资决策框架、商业逻辑 |
| Drucker | 管理/组织 | 管理原则、组织效能 |
| Paul Graham | 创业/技术 | 创业思维、技术趋势 |
| Linus | 技术/系统 | 工程思维、系统设计 |
| Martin | 设计/产品 | 产品思维、用户体验 |
| Steve Jobs | 创新/营销 | 创新方法、产品哲学 |
| Feynman | 科学/学习 | 学习方法、科学思维 |
| Sophi | 综合/情感 | 人文关怀、情绪智慧 |

每个 Agent 从自己视角提炼精进和突破，最后整合成统一报告。

## 存储规范

每日精进输出保存到：
- 文件：`memory/growth/YYYY-MM-DD.md`
- 索引：`memory/growth/INDEX.md`（按月汇总关键突破）

## 进阶用法

### 周回顾
每周日汇总本周所有每日精进，生成：
- 本周核心主题
- 最重要的3个认知突破
- 下周重点方向

### 月复盘
每月1日生成月度认知升级报告：
- 认知地图更新
- 能力圈变化
- 思维模式演化

### 主题聚合
按主题维度汇总（如"投资决策"、"产品思维"）：
- 该主题下的所有精进
- 认知演进轨迹
- 实践反馈循环
